# Implementation Plan: Screen Melt Transition

**Goal:** Implement the DOOM-style screen melt transition in `main_sdl.c` when the game state changes strictly from `SFG_GAME_STATE_MENU` to `SFG_GAME_STATE_LEVEL_START`.

## AS-IS vs TO-BE

### AS-IS (`main_sdl.c`)
```c
void mainLoopIteration(void)
{
  SDL_Event event;
  // ... events ...

  if (!SFG_mainLoopBody())
    running = 0;

  SDL_UpdateTexture(texture,NULL,sdlScreen,
    SFG_SCREEN_RESOLUTION_X * sizeof(uint16_t));
  SDL_RenderClear(renderer);
  SDL_RenderCopy(renderer,texture,NULL,NULL);
  SDL_RenderPresent(renderer);
}
```

### TO-BE (`main_sdl.c`)
```c
// New global state for wipe
static uint16_t wipe_scr_start[SFG_SCREEN_RESOLUTION_X * SFG_SCREEN_RESOLUTION_Y];
static uint16_t wipe_scr_end[SFG_SCREEN_RESOLUTION_X * SFG_SCREEN_RESOLUTION_Y];
static int wipe_y[SFG_SCREEN_RESOLUTION_X];

void mainLoopIteration(void)
{
  SDL_Event event;
  // ... events ...

  uint8_t stateBefore = SFG_game.state;
  
  // 1. Snapshot the starting frame if we are in the MENU
  if (stateBefore == SFG_GAME_STATE_MENU) {
      memcpy(wipe_scr_start, sdlScreen, sizeof(sdlScreen));
  }

  // 2. Tick the engine (this draws the new frame to sdlScreen)
  if (!SFG_mainLoopBody())
    running = 0;

  // 3. Detect the specific edge
  if (stateBefore == SFG_GAME_STATE_MENU && SFG_game.state == SFG_GAME_STATE_LEVEL_START) {
      // Snapshot the ending frame
      memcpy(wipe_scr_end, sdlScreen, sizeof(sdlScreen));
      
      // Initialize the randomized melt columns
      wipe_y[0] = -(rand() % 16);
      for (int i = 1; i < SFG_SCREEN_RESOLUTION_X; i++) {
          wipe_y[i] = wipe_y[i-1] + (rand() % 3) - 1;
          if (wipe_y[i] > 0) wipe_y[i] = 0;
          else if (wipe_y[i] == -16) wipe_y[i] = -15;
      }

      // Blocking Melt Loop
      int done = 0;
      while (!done && running) {
          done = 1;
          for (int i = 0; i < SFG_SCREEN_RESOLUTION_X; i++) {
              if (wipe_y[i] < 0) {
                  wipe_y[i] += 2; // Initial drop delay
                  done = 0;
              } else if (wipe_y[i] < SFG_SCREEN_RESOLUTION_Y) {
                  int dy = (wipe_y[i] < 16) ? wipe_y[i] + 1 : 8; // Accelerate drop
                  if (wipe_y[i] + dy >= SFG_SCREEN_RESOLUTION_Y) dy = SFG_SCREEN_RESOLUTION_Y - wipe_y[i];
                  wipe_y[i] += dy;
                  
                  // Reveal new screen (stationary) at top
                  for (int y = 0; y < wipe_y[i]; y++) {
                      sdlScreen[y * SFG_SCREEN_RESOLUTION_X + i] = wipe_scr_end[y * SFG_SCREEN_RESOLUTION_X + i];
                  }
                  // Drop the old screen downwards
                  for (int y = wipe_y[i]; y < SFG_SCREEN_RESOLUTION_Y; y++) {
                      sdlScreen[y * SFG_SCREEN_RESOLUTION_X + i] = wipe_scr_start[(y - wipe_y[i]) * SFG_SCREEN_RESOLUTION_X + i];
                  }
                  done = 0;
              }
          }
          
          // Pump events to prevent OS freeze during wipe
          while (SDL_PollEvent(&event)) {
              if (event.type == SDL_QUIT) { running = 0; done = 1; }
          }
          
          // Push to screen
          SDL_UpdateTexture(texture,NULL,sdlScreen, SFG_SCREEN_RESOLUTION_X * sizeof(uint16_t));
          SDL_RenderClear(renderer);
          SDL_RenderCopy(renderer,texture,NULL,NULL);
          SDL_RenderPresent(renderer);
          SDL_Delay(10); // Throttle the melt speed
      }
  }

  // Normal frame push
  SDL_UpdateTexture(texture,NULL,sdlScreen, SFG_SCREEN_RESOLUTION_X * sizeof(uint16_t));
  SDL_RenderClear(renderer);
  SDL_RenderCopy(renderer,texture,NULL,NULL);
  SDL_RenderPresent(renderer);
}
```

## Architectural Rationale (Minimal Path Strategy)
- **No Engine Modifications:** We strictly isolate the visual effect inside `main_sdl.c`. The `SFG_mainLoopBody()` engine does not need to know a wipe is happening.
- **Row-Major Memory:** Instead of replicating DOOM's complex `colMajorXform`, we iterate sequentially down the columns. While row-major columns cause cache misses, a modern CPU can easily handle 640x480 array access in real-time, making the code much shorter and less invasive.
- **State Trapping:** The loop temporarily traps execution until the melt reaches the bottom of the screen, pumping only the `SDL_QUIT` events to prevent the window manager from marking the app as unresponsive.
